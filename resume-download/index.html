Application
Health Check

The health check application is to check applications on servers and websites automatically for errors. The errors include errors and warnings generated by applications, elements missing on websites, and incorrect content of element on websites. The application developed in this project is able to check on applications, store the checking results in the database, and show the results through web pages. With the help of this program, owners, users, or developers can monitor servers and websites efficiently without spending much energy and time. When errors happen frequently or critical errors happened, the maintaining of the application can be done quickly.





There are three main parts in this application including the checking part that conducts check on objects, database that stores those checking results, and the frontend for showing the results. For the checking component, the parameters need to be load from a source instead of hard coding all the test cases. Then it can check the objects and generate errors if the cases failed, and the results need to be stored in database. In terms of the database, the test cases and the test results all need to be able to be stored. In addition, for the frontend, results in the database need to be able to be loaded and viewed.
